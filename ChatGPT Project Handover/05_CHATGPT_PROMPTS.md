# ChatGPT Continuation Prompts

Use these prompts inside a new ChatGPT Project if you want to continue the work quickly.

## Prompt 1: Continue the website build

```text
You are continuing a static site project called Backyard Upgrades.

Read these files first:
- 00_START_HERE.md
- 01_PROJECT_SUMMARY.md
- 02_SITE_STRUCTURE.md
- 03_CONTENT_INVENTORY.md
- 04_PUBLISHING_WORKFLOW.md

Then inspect the working site folder and help me continue building it.

Priorities:
- keep the premium editorial look
- preserve the static no-build setup unless there is a strong reason to change it
- prepare the site for GitHub hosting
- do not remove the existing article content
```

## Prompt 2: Review and upgrade affiliate links

```text
Continue the Backyard Upgrades project.

Inspect the article Markdown files and the article renderer. The Package 2 upgrade already added affiliate-tagged Amazon links, but some may still be search-based rather than exact product pages.

Please help me review and upgrade those links where possible while preserving the current affiliate card styling and premium visual direction.
```

## Prompt 3: Add SEO and publishing pages

```text
Continue the Backyard Upgrades static site project.

The legal pages and crawl files already exist.

Please audit them, improve any weak copy, and add the next layer of launch polish such as schema, stronger metadata, favicon assets, and internal linking.
```

## Prompt 4: Prepare GitHub deployment

```text
Continue the Backyard Upgrades project and prepare it for GitHub.

Please:
- create a clean repo-ready structure
- add a strong README
- add a .gitignore
- confirm the static asset paths are safe for GitHub hosting
- review `robots.txt` and `sitemap.xml` for the final host
- explain the easiest deployment path for GitHub Pages
```

Continue the Backyard Upgrades project.

I want to expand the article library while preserving the same tone: practical, clear, trustworthy, and premium.

Please propose the next 20 article ideas by category, identify the best affiliate opportunities, and then generate the next batch in the same site structure.
